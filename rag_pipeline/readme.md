## Task 1.6 — Skeleton RAG Knowledge Base

### Objective
Build the initial RAG knowledge base and FAISS vector index from the URFD dataset
to enable knowledge-grounded fall detection .

### What was done
- Loaded pose feature JSONs generated by Ram's `pose_features.py` (70 sequences —
  30 FALL + 40 ADL) and extracted key metrics per sequence: `torso_vertical_diff_avg`,
  `torso_vertical_diff_min`, `hip_y`, `shoulder_y`, `nose_y`
- Built 70 structured `case_entry` dicts with rich natural language descriptions
  derived from actual pose values — fall cases describe rapid collapse and floor
  contact; ADL cases explicitly contrast by noting body never becomes horizontal
- Encoded all 70 cases using `sentence-transformers/all-MiniLM-L6-v2` (384-dim
  embeddings) and built a FAISS `IndexFlatL2` vector index
- Validated retrieval with 3 test queries — system correctly retrieves FALL cases
  for collapse queries and NO_FALL cases for ADL queries
- Saved final deliverables: `cases.json` (70 annotated cases) and `faiss.index`
  (70 vectors, 384-dim, ~105KB)

### Deliverables
- `rag_pipeline/faiss.index` — FAISS L2 index, 70 vectors, 384 dimensions
- `rag_pipeline/rag_pipeline.ipynb` — full pipeline notebook
- `cases.json` — 70 annotated cases (30 FALL + 40 NO_FALL) with pose summaries

### Tools & Libraries
- `sentence-transformers` — all-MiniLM-L6-v2 for text embeddings
- `faiss-cpu` — vector similarity search
- `mediapipe` pose JSONs from Ram's pipeline as input
